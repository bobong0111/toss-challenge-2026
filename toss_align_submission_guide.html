호스트 여정 상세 사양
단계 화면 및 명칭 상세 구현 작동 스펙
1단계 일정 홈 (달력) 사내 SSO 원터치 로그인 후 진입하며, 주간 캘린더에서 개인 일정을 관리합니다.
2단계 호스트 대시보드 "디자인1팀 회의" 퀵 템플릿 선택, 'ㄱㅌㅅ' -> '김토스' 지능형 초성 동료 검색 및 추가를 수행합니다.
3단계 요청 전송 히트맵을 확인하여 "모두에게 물어보기" 요청을 발송한 뒤 캘린더 홈으로 자동 이동합니다.
4단계 취합 알림 대기 3.5초 뒤 보라색 테마의 "디자인1팀 회의 취합 완료 응답 도착" 팝오버 알림이 누적 노출됩니다.
5단계 최종 확정 팝오버 클릭 시 결과창으로 이동, 고정 크기(312x108px)의 100% 추천 시간을 클릭해 회의를 최종 확정합니다.
참여자 여정 상세 사양
단계 화면 및 명칭 상세 구현 작동 스펙
1단계 알림 수신 (홈) 우측 하단에 상시 떠 있는 "이토스님의 회의 요청" 파란색 팝오버 알림을 클릭해 입장합니다.
2단계 조율방 (참석자) 개인 약속이 투영된 달력 상에서 이토스 제안 슬롯 중 참석 선호 시간대를 클릭해 표기합니다.
3단계 응답 전송 '이 시간으로 응답' 클릭 후 파란색 체크박스 이미지(`check.png`)와 완료 문구 모달 피드백을 수신합니다.
4단계 확정 알림 대기 홈 복귀 후 4.5초 뒤 "디자인&개발 협업 미팅 최종 확정" 알림 카드가 우측 하단에 도착합니다.
5단계 캘린더 추가 "내 캘린더에 추가 →" 링크 클릭 시 금요일 10시 일정 블록이 자신의 달력 해당 구역에 자동 바인딩됩니다.

상태 격리 및 복원 프로세스 (Data Isolation & Reversion)
본 시스템은 호스트 모드의 멤버 데이터(`p2`: 윤디자, `p3`: 박디자 등)와 참여자 모드의 멤버 데이터(`p2`: 박개발, `p3`: 최마케터 등)가 전역 ID를 공유하여 사용합니다. 팝오버를 연속적으로 오가며 화면을 롤링할 때 전역 변수가 오염되어 6/6 빈 시간대(금요일 13시)가 깨지는 현상이 존재했습니다.
이를 방지하기 위해 대시보드 전송점, 취합 팝오버 클릭 시점 및 홈 복귀점에 `restoreScheduleData()` 헬퍼를 완전 바인딩하여 데이터 누출과 오염을 차단했습니다.
// app.js에 설계된 일정 복원 아키텍처 function restoreScheduleData() { // 로컬 스토리지 toss_schedule_data 세이브 로드 시도 if (!loadScheduleDataFromLocalStorage()) { initializeDefaultScheduleData(); // 실패 시 호스트 프리셋 로드 } }
세로 셀 병합 알고리즘 (Vertical Spanning)
연속된 동일 범주의 약속이나 연차가 달력 칸칸이 쪼개지지 않고 깔끔한 단일 카드로 그려지도록 드로잉 모듈을 구축했습니다. 병합 대상 첫 슬롯에 `rowspan` 속성을 동적으로 늘려 병합하고, 가려진 아래 칸들은 루프에서 렌더링을 패스 처리하는 알고리즘으로 설계되었습니다.
점심시간 비활성화 영역 (Lunch-Time Lockout)
매일 `12:00 - 13:00` 시간대는 기본 테이블의 CSS 클래스를 `status-lunch`로 통일하고 마우스 클릭 및 포커스 영역에서 완전 차단(`pointer-events: none`)하여 일정이 잘못 입력되는 케이스를 설계 레벨에서 원천 예방했습니다.
